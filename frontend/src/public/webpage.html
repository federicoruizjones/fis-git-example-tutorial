¿Qué es Git?

Git es un sistema de control de versiones (VCS) de código distribuido. Podemos destacar las siguientes características:

  • Rápido y simple
  • Sistema de trabajo con ramas
  • No depende de un repositorio central
  • Muy potente
  • Flexibilidad

¿Para qué sirve Git?

Git permite manejar proyectos muy grandes y distribuidos. Es decir, los desarrolladores pueden colaborar y realizar cambios
en proyectos compartidos, a la vez que mantienen un seguimiento detallado de su progreso.

¿Qué es un repositorio?

Un repositorio es la ubicación o ruta en la que se almacena toda la información de un proyecto como imágenes, código,
carpetas, documentos, etc. Cada proyecto cuenta con su propio repositorio único, por lo que la ruta de acceso será
exclusiva para el proyecto.

¿Qué es una rama?

En el caso que queramos trabajar en una parte concreta de nuestro proyecto de forma aislada sin afectar al repositorio principal,
tendremos que hacerlo mediante una rama (branch).
El branch creará una copia exacta de nuestro proyecto.

¿Qué es un conflicto?

Un conflicto ocurre cuando dos o más desarrolladores están editando el mismo espacio de código ya que, en estos casos,
Git no puede determinar automáticamente qué es correcto.
Para evitar esta situación, los desarrolladores trabajan en ramas independientes.

¿Cómo se resuelve un conflicto?

Los conflictos se deben resolver mergeando los archivos en cuestión de manera manual. Es decir, comparar los archivos
y determinar cuál es el correcto o combinarlos.

Git init

git init permite inicializar un repositorio a partir de una carpeta donde tengas almacenados los archivos que quieras gestionar.

Git clone

git clone permite obtener una copia local de un repositorio remoto.

Git status

git status permite conocer la rama en que te encuentras, el estado de tus archivos y si es necesario actualizar el servidor remoto.

Git add

git add permite agregar uno o varios archivos al stack de archivos prontos para el próximo commit.

Git commit

git commit permite confirmar los cambios realizados en el proyecto.

Git log

git log permite ver el historial de commits en orden cronológico inverso. Es decir, comienza por el último commit realizado.

Git remote

git remote permite ver todos los repositorios remotos.

Git pull

git pull fusiona todos los cambios que se han hecho en el repositorio remoto con el directorio de trabajo local.

Git push

git push se usa para enviar confirmaciones locales a la rama maestra del repositorio remoto.

Git checkout

git checkout crea ramas y te ayuda a navegar entre ellas.

Git branch

git branch permite listar, crear o eliminar ramas. Por ejemplo, si queremos ver todas las ramas usamos git branch o si
queremos elimiar una usamos git branch -d [name].

Git merge

git merge se usa para funcionar una rama con otra rama activa.
